git在vscode中的使用

（好多坑啊。。。。哭泣
（填坑填坑！！记下来，不然之后绝对要忘记

首先呢，按照网上的教程，一步一步来
这里，先来个概述

  • 配置环境变量：在我的电脑配置环境变量，添加git的path
  • vscode安装插件：在vscode2017之后就不用安装git的插件了，自带的
  • 本地仓库捣鼓一番
  • 创建远程仓库
  • 本地仓库发布到远程仓库

1.配置环境变量

  1. 找到git安装的位置，然后找到git的cmd文件，把这个文件所在的路径,添加到电脑的环境变量
  2. 并且在vscode的用户设置中设置好git.path
    如图，

2.vscode 安装插件

我的vscode版本是2017的，所以不用安装网上说的那个插件（不过我搜了一下拓展商城，貌似没有找到╥﹏╥…哈哈那就先不管啦~

3.本地仓库捣鼓一番

就如上几篇说的那样来操作啦，具体操作可以看上几篇~~
这里来粗略地复习一下（敲黑板，咳咳

  • 初始化仓库 git init
  • 添加文件，并将其放到仓库门口：git add ./
  • 将文件从仓库门口放到房间里：git commit -m "说明"

4.创建远程仓库

就是在GitHubt账户头像附近 点击 + new repository
创建成功之后就复制仓库地址，下面用来连接本地仓库

5.本地仓库发布到远程仓库

连接远程仓库：
git remote add origin git@github.com:fearlessxmm/test113.git
这时候，我们来到vscode中，看到下面的图，具体操作图中展示了

然后,在vscode 中 add commit一条龙之后，点击push（推送）

but！！！发现我的vscode提示“把本地库内容推送到远程库出现错误”（哭了，这里找了好多好多资料都没有这个提示，别人都是一气呵成的╥﹏╥…

后来后来，找了好久，终于找到一个，就试了一下：就是在git文件夹的config文件中修改url

实际上是这样的，一开始我是用之前连接了远程仓库的本地仓库来做这一系列的工作，可能有什么冲突吧，git Bash上是可以push的，但是到了vscode就不行。

但是呢，其实只要做了上面连接远程仓库的命令之后，就已经在git文件夹config文件中创建了连接的url了，如图，

总结一点：
连接远程仓库的命令之后，就已经在git文件夹config文件中有url，并且这个url是你要连接的仓库没错的话，就可以在vscode中执行push等等操作了。
